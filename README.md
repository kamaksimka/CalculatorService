# CalculatorService

Простой веб-сервис-калькулятор на ASP.NET Core, упакованный в Docker.

## Возможности

- Базовые арифметические операции: сложение, вычитание, умножение, деление
- REST API интерфейс
- Поддержка Docker и Docker Compose для простого развертывания

## Технологии

- .NET Core / ASP.NET Core
- C#
- Docker
- Docker Compose

## Детали реализации
Данный сервис может принимать запросы как по REST API через HTTP запросы так и с помощью gRPC. Для обработки запросов независимо от того откуда они пришли используется класс `CalculatorEngine`.
В конструкторе он принимает фабрику команд и список инструкция. Паттерн команда, позволяет отделить реализацию комманд от их вызова, а также инкапсулировать код каждой команды. А паттерн фабрики для создания команд позволяет абстрагироваться от
создания команд, что в дальнейшем могло бы облегчить добавление новых операций не меняя класс `CalculatorEngine`.

### Алгоритм работы CalculatorEngine

#### 1. Инициализация
- Принимает фабрику команд (`ICommandFactory`) и список инструкций.
- Разделяет инструкции на:
  - **calc**: хранятся в `_calcInstructions` (ключ — имя переменной).
  - **print**: имена переменных сохраняются в `_printVariables`.
- Проверяет уникальность переменных в `calc` — при дублировании выбрасывает исключение.

#### 2. Выполнение (`ExecuteAsync`)
- Запускает параллельное разрешение только переменных, указанных в `print` не тратя ресурсы на вычисление лишнего.
- Возвращает список результатов (`PrintResult`).

#### 3. Разрешение переменной (`ResolveVar`)
- Использует `ConcurrentDictionary` для кэширования задач — одна переменная вычисляется единожды.
- Находит инструкцию `calc` для переменной.
- Параллельно получает значения левого и правого операнда (`GetValue`).
- Создает команду через фабрику на основе оператора (`instr.Op`) и выполняет её.

#### 4. Получение значения операнда (`GetValue`)
- Если операнд — число (`long`), возвращает его.
- Если операнд — имя переменной, рекурсивно вызывает `ResolveVar`.
- Кэширование на уровне `ResolveVar` предотвращает повторные вычисления одной переменной.

#### 5. Ключевые особенности
- **Параллелизм**: левый и правый операнды вычисляются одновременно.
- **Кэширование задач**: одна и та же переменная вычисляется только один раз, даже если запрошена многократно.
- **Рекурсивное разрешение**: поддерживаются выражения с зависимостями между переменными.

## Запуск с Docker
Соберите и запустите контейнер: `docker-compose up --build`

## Примеры запросов и ответов

### Test 1: Простейшее сложение
Проверка базовой функциональности сложения двух констант.

**Запрос:**
```json
POST /api/calculator/execute
Content-Type: application/json

[
  {
    "type": "calc",
    "op": "+",
    "var": "x",
    "left": "1",
    "right": "2"
  },
  {
    "type": "print",
    "var": "x"
  }
]
```

**Ответ**:
```json
Status: 200 OK
{
  "items": [
    {
      "var": "x",
      "value": 3
    }
  ]
}
```

### Test 2: Зависимые переменные
Проверка цепочки вычислений, где результат одной операции используется в следующей.
**Запрос:**
```json
POST /api/calculator/execute
Content-Type: application/json

[
  {
    "type": "calc",
    "op": "+",
    "var": "x",
    "left": "10",
    "right": "2"
  },
  {
    "type": "print",
    "var": "x"
  },
  {
    "type": "calc",
    "op": "-",
    "var": "y",
    "left": "x",
    "right": "3"
  },
  {
    "type": "calc",
    "op": "*",
    "var": "z",
    "left": "x",
    "right": "y"
  },
  {
    "type": "print",
    "var": "w"
  },
  {
    "type": "calc",
    "op": "*",
    "var": "w",
    "left": "z",
    "right": "0"
  }
]
```

**Ответ:**
```json
Status: 200 OK
{
  "items": [
    {
      "var": "x",
      "value": 12
    },
    {
      "var": "w",
      "value": 0
    }
  ]
}
```
